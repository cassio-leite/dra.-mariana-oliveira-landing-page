# Frontend Rules

## Objetivo

Construir interfaces modernas, elegantes, acessíveis e fáceis de manter.

Aparência desejada:

- Premium
- Minimalista
- Sofisticada
- Responsiva
- Acolhedora
- Com bastante espaço em branco

---

# Stack

- React
- TypeScript
- Vite
- Tailwind CSS
- shadcn/ui
- Framer Motion

---

# Componentização

- **SEMPRE** crie componentes pequenos e reutilizáveis.
- **SEMPRE** extraia código repetido para componentes ou funções utilitárias.
- **EVITE** duplicação de código.
- **PRIORIZE** composição em vez de componentes gigantes.
- **NUNCA** misture responsabilidades diferentes em um mesmo componente.

---

# TypeScript

- **SEMPRE** utilize TypeScript.
- **SEMPRE** tipar props, estados e funções.
- **EVITE** usar `any`.
- **PRIORIZE** tipos explícitos.

---

# Tailwind CSS

- **SEMPRE** utilizar Tailwind CSS.
- **SEMPRE** reutilizar classes quando houver repetição.
- **EVITE** classes excessivamente longas.
- **PRIORIZE** legibilidade.

---

# shadcn/ui

- **SEMPRE** reutilizar componentes do shadcn/ui.
- **NUNCA** recriar componentes já existentes.
- **PRIORIZE** Button, Card, Accordion, Sheet, Dialog, Badge e Separator.

---

# UI e Design

- **PRIORIZE** aparência premium.
- **PRIORIZE** muito espaço em branco.
- **PRIORIZE** hierarquia visual clara.
- **PRIORIZE** layouts elegantes e modernos.
- **EVITE** interfaces poluídas.
- **NUNCA** adicionar elementos desnecessários.
- **NUNCA** utilizar efeitos exagerados.

---

# Responsividade

- **SEMPRE** seguir abordagem mobile-first.
- **SEMPRE** verificar desktop, tablet e mobile.
- **PRIORIZE** boa experiência em dispositivos pequenos.

---

# Animações

- **PRIORIZE** animações suaves e discretas.
- **SEMPRE** utilizar Framer Motion.
- **SEMPRE** preferir Blur Fade para elementos importantes.
- **NUNCA** exagerar em animações.
- **EVITE** efeitos chamativos ou distrativos.

---

# Alterações

- **SEMPRE** fazer mudanças pequenas e incrementais.
- **SEMPRE** preservar componentes não relacionados.
- **NUNCA** reescrever arquivos inteiros sem necessidade.
- **PRIORIZE** modificar apenas o necessário.

---

# Código

- **PRIORIZE** Clean Code.
- **PRIORIZE** legibilidade.
- **PRIORIZE** simplicidade.
- **EVITE** complexidade desnecessária.
- **EVITE** abstrações prematuras.

---

# Acessibilidade

- **SEMPRE** utilizar HTML semântico.
- **SEMPRE** adicionar atributos acessíveis quando necessário.
- **PRIORIZE** boa navegação por teclado.

---

# Performance

- **PRIORIZE** componentes leves.
- **EVITE** renders desnecessários.
- **NUNCA** adicionar dependências sem necessidade.

---

# Desenvolvimento com IA

- **SEMPRE** preservar a identidade visual existente.
- **SEMPRE** reutilizar componentes já criados.
- **SEMPRE** analisar a estrutura atual antes de implementar novas funcionalidades.
- **NUNCA** alterar código não relacionado à tarefa solicitada.
- **NUNCA** fazer grandes refatorações sem necessidade.
- **PRIORIZE** soluções simples e fáceis de manter.
